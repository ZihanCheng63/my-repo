# RuyiSDK IDE Eclipse 插件测试报告

## 一、测试概述
本次测试旨在验证 RuyiSDK IDE Eclipse 插件的核心功能，包括项目创建、交叉编译、虚拟环境管理及程序运行等集成能力，保障插件在 Eclipse IDE 中对 RISC-V 开发流程的集成支撑效果。

## 二、测试范围
- 项目创建
- 交叉编译
- 程序运行
- 虚拟环境管理

## 三、测试环境
- 操作系统：Fedora(x86_64)
- IDE：Eclipse IDE
- 插件版本：RuyiSDK Eclipse 插件
- 硬件配置：标准x86_64开发机
- 配套工具链：gnu-plct-0.20250912.0

## 四、测试用例执行情况
### TC01 项目创建
- **测试步骤**：通过 Eclipse 菜单手动创建项目，然后使用 RuyiSDK 插件模板创建项目
- **测试结果**：手动创建的项目无法被 Eclipse 识别为 “可构建项目”，右键菜单中缺少 Build Project 等关键选项
- **状态**：未通过

### TC02 交叉编译
- **测试步骤**：在终端手动调用 RuyiSDK 编译器执行编译命令，然后在 Eclipse 中尝试配置外部工具调用编译命令，验证插件一键构建功能
- **测试结果**：终端手动编译成功，生成 RISC-V 可执行文件 build/hello；Eclipse 中通过外部工具配置可触发编译
- **状态**：通过

### TC03 程序运行
- **测试步骤**：在终端通过 QEMU 运行编译后的程序，在 Eclipse 中尝试配置运行环境，验证插件一键运行功能
- **测试结果**：终端运行成功，输出预期结果 'Hello RuyiSDK!'，但 Eclipse 中无法直接通过插件一键运行，需手动配置外部工具
- **状态**：通过

### TC04 虚拟环境管理
- **测试步骤**：点击 Eclipse 顶部菜单栏 RuyiSDK → Venv (Virtual Environments)，尝试创建/激活虚拟环境
- **测试结果**：菜单存在，可以找到创建或激活虚拟环境的有效入口
- **状态**：通过

## 五、缺陷分析
- 插件项目创建功能存在缺陷，无专属项目模板，且无法识别手动创建的项目，缺少核心构建相关菜单选项，无法完成插件侧的项目初始化；
- 插件未实现一键构建、一键运行的集成化功能，需依赖Eclipse外部工具手动配置命令才能完成编译和运行操作。

## 六、测试结论
RuyiSDK IDE Eclipse 插件的底层配套编译、运行能力可正常使用，能支撑基础的RISC-V程序开发需求，但插件的核心集成化功能存在缺失，项目创建、一键构建等关键集成功能未实现或存在缺陷，需手动结合Eclipse外部工具完成操作，插件的集成体验和功能性未达到完善的使用标准，建议对缺失的集成功能进行开发和优化。
